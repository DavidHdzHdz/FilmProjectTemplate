# 00_PROJECT_OVERVIEW — Instrucciones

## ¿Qué es esta carpeta?

La visión general del proyecto: nombre, fase actual, equipo, cronograma y notas de alto nivel. Se llena **después de `01_CONCEPTO`** — necesitas la premisa y el logline para poder describir el proyecto con claridad.

---

## Orden de llenado

```
1. Project_Overview.md   ← resumen ejecutivo del proyecto
2. Timeline.md           ← cronograma de fases y fechas estimadas
```

---

## Archivo por archivo

**`Project_Overview.md`** — El resumen ejecutivo: nombre del proyecto, logline, premisa, género, tono, equipo y estado actual. Es el documento que le das a alguien para que entienda el proyecto en dos minutos.

> **Tips:**
> - No lo llenes antes de tener el logline y la premisa escritos en `01_CONCEPTO` — de lo contrario terminarás con una descripción vaga que tendrás que reescribir.
> - Actualízalo cada vez que el proyecto cambie de fase.

**`Timeline.md`** — Las fechas estimadas por fase: cuándo cierras el guion, cuándo empieza preproducción, cuándo es el rodaje, cuándo entregas el corte final.

> **Tips:**
> - Las fechas de las fases tempranas (concepto, personajes, escritura) son flexibles. Las de rodaje y entrega no.
> - Si trabajas en equipo, este archivo es el punto de referencia compartido.

---

## Tips generales

- Este no es el primer archivo que llenas — es el segundo, después de `01_CONCEPTO`.
- Revísalo al inicio de cada nueva fase para asegurarte de que el resumen sigue siendo preciso.
